# Quickstart — Ensembra in 5 minutes

This walks through installing Ensembra, configuring it, and running your first pipeline on a tiny sandbox project.

## 1. Install Claude Code

Ensembra is a Claude Code plugin. You need Claude Code installed first.

```bash
# macOS
brew install claude-code

# verify
claude --version
# expected: 2.x.x or higher
```

## 2. Clone Ensembra

```bash
git clone https://github.com/HotRedMat/ensembra
```

## 3. Validate the plugin

```bash
claude plugin validate ./ensembra
# expected: ✔ Validation passed
```

## 4. Create a tiny sandbox to play with

```bash
mkdir -p /tmp/ensembra-demo/src /tmp/ensembra-demo/tests
cd /tmp/ensembra-demo

cat > src/calculator.js <<'EOF'
function add(a, b) { return a + b; }
function subtract(a, b) { return a - b; }
module.exports = { add, subtract };
EOF

cat > tests/calculator.test.js <<'EOF'
const { add, subtract } = require('../src/calculator');
console.assert(add(2, 3) === 5, 'add failed');
console.assert(subtract(5, 3) === 2, 'subtract failed');
console.log('all tests passed');
EOF
```

## 5. Run your first Ensembra pipeline

```bash
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits \
  --print "/ensembra:run feature Add a multiply function to src/calculator.js that multiplies two numbers, and add a test for it. Follow the existing module.exports pattern and console.assert style. Keep the whole change minimal."
```

What happens:

1. **Phase 0 Gather** — Claude Code reads `src/calculator.js`, `tests/calculator.test.js`, looks for common utilities, scans git history (if any).
2. **Phase 1 Deliberate** — 6 performers discuss the change. Reuse-First policy ensures they prefer extending the existing pattern instead of introducing new files.
3. **Phase 2 Execute** — Claude Code edits `calculator.js` and `calculator.test.js`.
4. **Phase 3 Audit** — designated auditors verify the diff against the plan.
5. **Phase 4 Document** — scribe writes `docs/reports/tasks/YYYY-MM-DD-add-multiply.md`.

Verify the result:

```bash
node tests/calculator.test.js
# expected: all tests passed
```

## 6. Try other presets

```bash
# fix a bug
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits --print "/ensembra:run bugfix <description of the bug>"

# refactor with duplication extraction
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits --print "/ensembra:run refactor Extract the duplicated formatDate function to src/commons/"

# read-only security audit
claude --plugin-dir /path/to/ensembra --print "/ensembra:run security-audit Review src/login.js"

# read-only source analysis
claude --plugin-dir /path/to/ensembra --print "/ensembra:run source-analysis Check for design drift"
```

## 7. Generate a handover document

```bash
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits --print "/ensembra:transfer"
# creates docs/transfer/YYYY-MM-DD-project.md
```

Or scope it to a subdirectory:

```bash
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits --print "/ensembra:transfer src/payment"
```

## 8. Roll up reports

```bash
# daily
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits --print "/ensembra:report daily"

# weekly
claude --plugin-dir /path/to/ensembra --permission-mode acceptEdits --print "/ensembra:report weekly"
```

## 9. Configure (optional but recommended)

```bash
claude --plugin-dir /path/to/ensembra --print "/ensembra:config"
```

The interactive picker lets you change Performers, Presets, Rounds, Deep Scan items, Transports, Timeouts, Logging, Reports, and Reuse-First Policy — all without editing JSON by hand.

## 10. Optional: set up external transports

### Ollama (free, local)

```bash
ollama pull qwen2.5:14b llama3.1:8b
```

Ensembra will auto-route the security performer to `qwen2.5:14b` and the qa performer to `llama3.1:8b`. If Ollama is unreachable, performers fall back to Claude sub-agents.

### Gemini (free tier)

Claude Code prompts you for the Gemini key when the plugin is installed or enabled — no manual file editing needed.

```bash
claude plugin install ensembra
# Claude Code asks for gemini_api_key (input hidden, stored in OS keychain)
```

Already installed without a key? Re-trigger the prompt:

```bash
claude plugin disable ensembra
claude plugin enable ensembra
```

Get a free key at <https://aistudio.google.com/app/apikey>. The architect performer will route to `gemini-2.5-flash` when the key is set; otherwise it falls back to a Claude sub-agent.

## Troubleshooting

**Plugin doesn't load** — run `claude plugin validate /path/to/ensembra` to see structural errors.

**Skills don't appear** — make sure you're using `--plugin-dir` and the path is absolute.

**Rework loop hits the limit** — the default is 2 attempts. Raise it via `/ensembra:config → Rounds → Rework 상한` if your task is genuinely complex.

**Pipeline halts at low consensus** — that's the safety feature. Read the rejection summary, decide whether the request is too broad, and either split it or override via `/ensembra:config → Rounds → 중단 합의율` (not recommended).
