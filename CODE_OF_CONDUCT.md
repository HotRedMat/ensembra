# Code of Conduct

## Short version

Be kind. Assume good faith. Give specific technical feedback, not personal judgments. If you disagree with a design decision, cite `CONTRACT.md` or `INTERVIEW.md` and explain why the current rationale no longer holds.

## The rules

1. **Respect the design boundaries.** Do not repeatedly push for changes that have been explicitly excluded (ChatGPT integration, session handoff). Those decisions are documented; if you want to revisit them, start a discussion with new evidence.

2. **Critique code, not people.** "This function has a race condition on line 42" is fine. "You don't understand concurrency" is not.

3. **No harassment, hate speech, or personal attacks.** This applies to issues, PRs, discussions, commit messages, and any other project-adjacent space.

4. **No secrets in public.** Never paste API keys, tokens, passwords, or `.env` contents into any public space. If you accidentally do, rotate the credential immediately and notify the maintainer.

5. **Minimize noise.** Before filing an issue or opening a PR, search for existing discussions. Before asking a question, check the docs (`README.md`, `CONTRACT.md`, `CHANGELOG.md`).

6. **English and Korean are both welcome.** Ensembra's design documents are primarily in Korean; English issues and PRs are fine. Be patient when there's a translation gap.

## Enforcement

The project maintainer (Seungho Lee, misstal80@gmail.com) has final authority on moderation decisions. Consequences for violations range from a warning to a permanent ban, depending on severity and repetition.

Actions taken include:
- **Warning** for minor or first-time incidents
- **Comment lock** on the specific thread
- **Temporary ban** from the repository (1–30 days)
- **Permanent ban** for severe or repeated violations

## Reporting

To report a violation, email `misstal80@gmail.com` with:
- Links to the problematic content
- A short description of what happened
- Any prior context the maintainer should know

Reports are handled confidentially. Do not post accusations in public issues.

## Scope

This Code of Conduct applies to all project-managed spaces: the GitHub repository, issues, discussions, PRs, the plugin marketplace entry, and any official Ensembra communication channels. It does not extend to personal accounts, unrelated projects, or off-platform interactions unless they directly spill into the project.
